User Guide

This tool processes oral history interview transcripts through a multi-step AI pipeline to produce structured metadata for the Smithsonian Civil Rights History Project archive.

The pipeline turns a transcript into structured metadata for archives, web applications, and demos. It builds text blocks, labels topics, generates chapters and summaries, then optionally scores engagement and extracts clips.

Batch processing uses the same configuration, so a clean first run on the Upload page becomes the source of truth for the rest of the pipeline.

Pipeline steps

1. Upload
Upload an SRT transcript file and set your OpenAI API key. Optional modules (Questions, Engagement, Clips) can be toggled here.
2. Labeling
The transcript is split into blocks and each block is labeled with a topic category and subtopics by the AI.
3. Table of Contents
Labeled blocks are merged into a hierarchical table of contents with timestamps. You can edit the raw JSON before continuing.
4. Chapterization
Topic transitions are detected to divide the interview into chapters. Chapter boundaries can be adjusted manually.
5. Summarization
An overall summary and per-chapter summaries are generated. System and user prompts are fully editable before running.
6. Questions
Suggested research questions are generated from the interview content. Optional — enable on the Upload page.
7. Tuning
Summaries are scored against quality and accuracy thresholds. Failing summaries are automatically regenerated up to the configured retry limit.
8. Engagement
Each chapter is scored for public engagement potential. Optional — enable on the Upload page.
9. Clips
Key excerpts are identified as candidate video clips with start/end timestamps. Optional — enable on the Upload page.
10. Results
All outputs are bundled into a downloadable results.json file ready for archival ingest.

Dev mode

Run the full pipeline with mock data — no API key required and no tokens consumed. Useful for testing the UI and reviewing output formats.

Tips

  • If a step produces unexpected output, expand the Advanced section to edit the prompts before re-running.
  • The Pipeline metrics button in the header shows elapsed time and token usage per step.
  • Chapter boundaries on the Chapterization page can be adjusted in the raw JSON before continuing.
  • Use Start over on the Results page to clear the session and process a new interview.
